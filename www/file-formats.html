Output file formats (v2.0.3)

NOTE: Tab file outputs contain redundant information with the .aligned.bam and .vcf outputs. They will be deprecated in the near future.

$prefix.aligned.tab

A file containing all reads that were aligned. This file contains three two types of lines:
  • STR alignment lines: these give alignments of STR-containing reads. This includes single-end and members of paired-end reads aligned to an STR, as well as members of paired-end reads that have been stitched together. This also includes reads that only partially span an STR, in which case the allele reported gives a lower bound on the true allele represented by the read. Note that the partially spanning reads feature is in beta version and often produces unreliable allele reports. These reads are not currently used in any downstream steps.
  • Mate alignment lines: these give alignments of the mates of STR-containing reads. For paired-end alignment, only STR alignments of reads whose mate pairs also align in the same region are returned to ensure alignment specificity. For these lines in the alignment files, all fields specific to STR-aligned reads are omitted and replaced with "-".
This file has the fields:
  1. Read ID
  2. Nucleotides
  3. Quality score
  4. STR region
  5. STR period
  6. STR motif
  7. Chrom
  8. STR start
  9. STR end
  10. Read start
  11. Difference from reference (# bp)
  12. Reference copy number (# bp)
  13. Strand (1=reverse, 0=forward)
  14. CIGAR score for alignment
  15. 0/1 flag to indicate whether the read is partially spanning
  16. Alignment quality score (given as the sum of quality scores of bases mismatching the reference).
  17. 0/1 flag to indicate whether this line gives the alignment of the mate of an STR-containing read
  18. Distance between mate pairs (for paired-end read mode only. For single ends this field is set to -1.)
  19. 0/1 flag to indicate whether this alignment is the result of stitching two members of a mate pair together

$prefix.aligned.bam

A bam file containing all aligned reads. This file contains the same information as the aligned.tab file. This is in the standard bam format, with the following added fields:
  • XS: start position of matching STR
  • XE: end position of matching STR
  • XR: STR repeat
  • XC: reference copy number
  • XD: nucleotide length difference compared to reference
  • XX: 0/1 flag to indicate stitching of paired end reads
  • XP: 0/1 flag to indicate if an alignment is partially spanning.
  • XM: distance between mate pairs.
  • XN: (only if given) name of the STR repeat
  • XG: The extracted STR region
  • XQ: map quality score of the aligned read
  • RG: (only if given) read group
Note the bam output is not sorted.

$prefix.genotypes.tab

A list of all markers covered. Information is reported about both fully spanning STR reads and partially spanning reads. Currently the partially spanning reads are not taken into account when determining the allelotype. Also note that if lobSTR cannot confidently report two allelesa at a diploid locus, one allele might be reported as "NA". Loci in which no fully spanning reads were aligned but partially spanning reads were report an allelotype of "NA,NA". This file contains the following fields:
  1. chromosome of matched STR
  2. start position of matched STR
  3. end position of matched STR
  4. STR repeat
  5. Period of STR repeat
  6. Reference copy number (TRF notation)
  7. Allelotype call (allele1, allele2) (given in # bp difference from reference)
  8. Coverage
  9. Number of reads supporting the reported genotype
  10. Number of reads conflicting with the reported genotype
  11. All alleles (given in # bp difference from reference:number of reads supporting that allele) that aligned to that locus separated by "/"
  12. Score (posterior probability of call). This field is -1 if no allelotype is reported.
  13. Allele 1 score (marginal posterior probability of the first reported allele). This field is set to -1 if allele 1 is not reported.
  14. Allele 2 score (marginal posterior probability of the first reported allele). This field is set to -1 if allele 2 is not reported.
  15. Coverage by partially spanning reads
  16. Largest lower bound on the allele number reported by a partially spanning read.
  17. List of alleles reported for partially spanning reads (same format as column 11).
  18. Number of stitched reads aligned to this locus

$prefix.vcf

Required fields

  • CHROM: chromosome of the STR
  • POS: start position of the STR. Note, since the start position is used, STRs with the same start coordinate but different motifs must be filtered for compatibility with downstream vcftools.
  • ID: this field is set to ".".
  • REF: gives the base in the reference at CHROM:POS.
  • ALT: contains a comma separated list of "<STRVAR:allele>" where allele is the number of base pairs different from reference. If any STR variations were reported.
  • QUAL: contains -10log10(1-genotype score). Missing data has QUAL -1.
  • FILTER: this field is left blank

INFO fields

  • AC: Allele count in genotypes, for each ALT allele in the same order as listed. Not present if no alternate alleles listd.
  • AN: Total number of alleles in called genotypes
  • END: the end coordinate of the STR
  • MOTIF: the STR repeat motif
  • REF: the reference copy number
  • VT: variant type, set to STR

FORMAT fields

  • ALLREADS: string giving all alleles of all reads seen (column 11 from genotypes.tab file). The format is allele1|readcount;allele2|readcount, etc.
  • ALLPARIALREADS: string giving alleles of all partially spanning reads. These allele values give an UPPER BOUND on the true allele, and should be treated as such, not as allelotype calls. Same format as ALLREADS.
  • DP: STR coverage
  • GB: reported allelotype given in bp difference from reference. Set to "./." for no call.
  • Q: Likelihood ratio score of the allelotype call. Gives the likelihood of the reported genotype divided by the sum of likelihoods of all considered genotypes.
  • PP: posterior probabilitiy of the call (if available)
  • PL: Phred-scaled genotype likelihoods. Given for each possible pair of alleles from the ALT field. If j and k are the indices of the alleles in the ALT field, the (F(j/k) = (k*(k+1)/2)+j)th field gives the likelihood of allelotype (allele j, allele k) as in the standard VCF format. Normalized to give likelihoods relative to that of the maximum likelihood genotype, which will have a phred-scaled likelihood of 0.
  • AML: allele marginal likelihood ratio scores. The score for each allele gives the sum of all likelihoods of genotypes containing that allele divided by the sum of likelihoods of all genotypes considered.
  • AMP: allele marginal posterior probabilities (if available).
  • GT: genotype, given as indices into the list in the ALT field as in the standard VCF format. Set to "./." if no call.
  • MP: Upper bound on maximum partially spanning allele. Set to "." if no partially spanning reads.
  • PC: Coverage by partially spanning reads
  • STITCH: Number of stitched reads
